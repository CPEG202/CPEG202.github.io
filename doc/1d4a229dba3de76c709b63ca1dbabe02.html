Introduction

In this project you will design a multiplexer, a decoder, an encoder, and a shifter using Verilog HDL. Instead of building the circuit using logic operators, you will learn to describe a circuit behaviorally according to the functionality you wish the circuit to perform.

Before you begin, you should:

  • Have Real Digital®’s Blackboard set up;
  • Have the Xilinx® Vivado WebPACK™ installed;
  • Be able to write combinational logic using logic operators in Verilog and implement them on an FPGA;
  • Get familiar with General Project Submission Policy;

After you’re done, you should:

  • Be able to implement combinational logic behaviorally using Verilog HDL;

Step 2: Design a Multiplexer

This project starts with designing a 4-1 2-bit bus multiplexer. Eight on-board slide switches will be used to provide the data inputs, two push buttons will be used as select signals, and LEDs 0 and 1 will be used to show the output of the multiplexer. Instead of implementing the multiplexer using logic operators, you are going to describe the circuit behaviorally using an always block and an if-else statement, or a case statement.

Declare a 4-to-1 2-bit Bus Multiplexer

Create a Verilog module called mux_4_1 with inputs I0, I1, I2, I3, Sel, and output Y as follows:

module mux_4_1 (
    input [1:0] I0,
    input [1:0] I1,
    input [1:0] I2,
    input [1:0] I3,
    input [1:0] Sel,
    output [1:0] Y
);

Description of Multiplexer in Verilog HDL

There are multiple ways to describe the logic of a multiplexer in Verilog. You are presented three ways to code the multiplexer behaviorally with Verilog HDL. However, you only need one in your code. You are encouraged to try out all three of the different ways to understand and experience the difference in Verilog syntax.

1. Using ?: Selection Operator

The first way to code a mux behaviorally is to use the ?: selection operator. This method is most analogous to the if statement. You can think of this statement as follows, assign Y to I0 if the statement in the parenthesis is true, else do what is after the colon.

assign Y = (Sel == 2'd0) ? I0 : (
                (Sel == 2'd1) ? I1 : (
                    (Sel == 2'd2) ? I2 : I3
                )
            );

2. Using an always Block With an if Statement

The second way to code a mux is by using an always block together with an if-else statement. However, as Y is assigned in an always block, it needs to be declared as register type reg. You cannot have any assign statements within an always block.

reg [1:0] Y;

always @ (Sel, I0, I1, I2, I3)
begin
    if (Sel == 2'd0)
        Y = I0;
    else if (Sel == 2'd1)
        Y = I1;
    else if (Sel == 2'd2)
        Y = I2;
    else
        Y = I3;
end

3. Using an always Block With a case Statement

The third way to code a mux is by using an always block together with a case statement.

reg [1:0] Y;

always @ (Sel, I0, I1, I2, I3)
begin
    case (Sel)
        2'd0:
            Y = I0;
        2'd1:
            Y = I1;
        2'd2:
            Y = I2;
        2'd3:
            Y = I3;
        default:
            Y = 2'd0;
    endcase
end

Step 3: Design a Binary Decoder

In this section you are going to design a 3:8 binary decoder. Bus I[2:0] is used as an input for 3:8 decoder and bus Y[7:0] is used to indicate the output of the decoder.

Declare a 3:8 Binary Decoder

Create a Verilog module called decoder_3_8 with inputs I and outputs Y as follows:

module decoder_3_8 (
    input [2:0] I,
    output [7:0] Y
);

Behavioral Description of 3:8 Binary Decoder in Verilog HDL

The most efficient way to describe the behavior of a decoder is to use a case statement in an always block.

reg [7:0] Y;

always @ (I)
begin
    case (I)
        3'd0:
            Y = 8'd1;
        3'd1:
            Y = 8'd2;
        3'd2:
            Y = 8'd4;
        3'd3:
            Y = 8'd8;
        3'd4:
            Y = 8'd16;
        3'd5:
            Y = 8'd32;
        3'd6:
            Y = 8'd64;
        3'd7:
            Y = 8'd128;
        default:
            Y = 8'd0;
    endcase
end

Step 4: Design a Priority Encoder

In this section, you are going to design a 4-input priority encoder. Bus I[3:0] will be used as data inputs. One bit, Ein, will act as the “Enable” signal. Bus Y[1:0] will show the encoded value of inputs, and two one bit outputs GS and Eout will show the group signal and enable output, respectively.

Declare a 4-Input Priority Encoder

Create another Verilog module called encoder with inputs I, Ein, and outputs Eout, GS, and Y as follows:

module encoder (
    input [3:0] I,
    input Ein,
    output [1:0] Y,
    output GS,
    output Eout
);

Behavioral Description of Priority Encoder in Verilog HDL

The most efficient way to describe the behavior of a priority encoder is to use if-else statement in an always block. As the priority encoder has three outputs, you will code the behavioral description for those three outputs in three always blocks.

reg [1:0] Y;
reg GS;
reg Eout;

always @ (I, Ein)
begin
    if(Ein == 1)
        Y = (I[3] == 1) ? 2'd3 : (
                (I[2] == 1) ? 2'd2 : (
                    (I[1] == 1) ? 2'd1 : 2'd0
                )
            );
    else
        Y = 2'd0;
end

always @ (I, Ein)
begin
    if (Ein == 1 && I == 0)
        Eout = 1'b1;
    else
        Eout = 1'b0;
end

always @ (I, Ein)
begin
    if (Ein == 1 && I != 0)
        GS = 1'b1;
    else
        GS = 1'b0;
end

Step 5: Design a Shifter

In this section, you are going to design a 4-input Shifter. Bus I[3:0] will be used as data inputs. Three other one bit inputs will act as control signals F, R, and D. Last, a one bit input, En will be used as the Enable signal. Bus Y[3:0] will show the output of the shifter.

Declare a Shifter

Create a Verilog module called shifter with inputs I, En, D, R, F and outputs Y as follows:

module shifter (
    input [3:0] I,
    input D,
    input R,
    input F,
    input En,
    output [3:0] Y
);

Behavioral Description of Shifter in Verilog HDL

Similar to previous steps, you will use if-else statement again to implement the shifter.

reg [3:0] Y;

always @ (I, Ein)
begin
    if (En == 0)
        Y = I;
    else
        if (R == 0)
            Y = (D == 0) ? {I[2:0], F} : {F, I[3:1]};
        else
            Y = (D == 0) ? {I[2:0], I[3]} : {I[0], I[3:1]};
end

Concatenation

In the shifter’s behavioral code, {A,B} is used to concatenate two groups of signals into a bus. For example, Y = {I[2:0], F} means Y[3:1] = I[2:0] and Y[0] = F.
